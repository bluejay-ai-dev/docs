<Update label="April 3rd, 2026" tags={["Improvement", "API"]}>
  ## Redesigned Workflows

  Workflows are a structured way to test your agent along a conversation path.

  - **Agent and user turns** — you define what the agent should say or do (so simulations can check it) and what the digital human says on each step.
  - **Branching** — **options** nodes capture different things the caller might do next (speech, DTMF, silence).
  - **Coverage** — each distinct path through the graph becomes its own digital human, so every branch gets exercised.
  - **Docs** — cookbook and API reference refreshed; older workflow endpoints are under **Deprecated**.

  [Create workflow](/api-reference/endpoint/create-workflow) · [Cookbook](/cookbook/workflows)
</Update>

<Update label="March 20th, 2026" tags={["New", "Simulations"]}>

  Bluejay now supports SMS-based simulations, enabling you to test text-based agent flows end-to-end. Configure SMS simulations the same way you configure voice simulations — define digital humans, set up custom metrics, and run batch evaluations against your SMS agent.

<Update label="March 15th, 2026" tags={["New", "Alerts"]}>
  ## Threshold alarms

  Set threshold-based alarms on any custom metric to get alerted when agent performance degrades. Define upper or lower bounds, choose your notification channel (Slack, email, or webhook), and Bluejay will trigger alerts automatically when production metrics cross your thresholds.

  Alarms work across both observability and simulation metrics, so you can catch regressions in production and in testing.

  [Read the docs](/key-concepts/alerts/overview)
</Update>

<Update label="March 14th, 2026" tags={["Performance", "Observability"]}>
  ## Faster call log evaluation

  We made significant performance improvements to the observability evaluation pipeline:

  - **3x faster evaluation** for call logs with custom metrics
  - **Parallel metric execution** — multiple custom metrics now evaluate concurrently instead of sequentially
  - **Reduced API latency** for the `/evaluate` and `/re-evaluate` endpoints by approximately 40%

  These improvements apply automatically to all existing observability configurations.

  [Read the docs](/monitor/observability/overview)
</Update>

<Update label="March 3rd, 2026" tags={["Improvement", "Workflows"]}>

<Update label="February 24th, 2026" tags={["New", "Simulations"]}>
  ## Community-based simulation runs

  You can now run simulations against an entire community of digital humans in a single batch. Previously, simulations ran against individual digital humans or manually selected groups. Community-based runs let you test your agent against a diverse, pre-configured population in one click.

  Combine communities with custom metrics to get aggregate performance scores across demographic segments, persona types, or behavioral profiles.

  [Read the docs](/key-concepts/communities/overview)
</Update>

<Update label="February 17th, 2026" tags={["Improvement", "Metrics"]}>
  ## Custom metric formulas

  Custom metrics now support formula-based definitions in addition to LLM-as-a-Judges. Define metrics using arithmetic expressions over existing metric scores, enabling composite metrics like weighted averages or pass/fail thresholds without writing evaluation prompts.

  Formula metrics evaluate instantly and do not consume LLM credits.

  [Read the docs](/key-concepts/custom-metrics/overview)
</Update>
