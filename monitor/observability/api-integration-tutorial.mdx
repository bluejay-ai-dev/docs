This tutorial explains the shape of a typical Bluejay observability integration. The goal is to help teams move production conversation data into Bluejay in a clean, repeatable way.

## what you need

- A Bluejay account and API key
- Access to production transcripts, audio, or metadata
- A place in your stack where completed calls or conversations can be forwarded to Bluejay

## how it is used

Most teams connect their runtime or post-call pipeline to the Bluejay evaluate endpoint, attach any useful metadata, and then review the resulting metrics, traces, and alerts inside Monitor.

## recommended flow

1. Capture production conversation data from your provider or backend.
2. Normalize the payload so the transcript, metadata, and identifiers are consistent.
3. Send the conversation to Bluejay through the observability API.
4. Review the evaluation output in dashboards, alerts, and downstream tooling.
