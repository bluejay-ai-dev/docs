  <div data-glossary-item data-term="Call Log"><Accordion title="Call Log">
A stored record of a production conversation along with its evaluation scores, latency metrics, and metadata. Call logs can be re-evaluated when custom metrics change.

  <div data-glossary-item data-term="CHIRP"><Accordion title="CHIRP">
Conversational Handoff for Inter-agent Realtime Protocol — the message envelope used for WebSocket-based simulations. CHIRP defines message types including text, audio, status, and control.

See more: [WebSocket Integration](/simulation-integrations/websockets)
</Accordion></div>
  <div data-glossary-item data-term="Community"><Accordion title="Community">
A reusable group of Digital Humans organized around a shared testing purpose. Communities enable consistent benchmarking across agents and simulation runs, and a Digital Human can belong to multiple communities.

See more: [Communities Overview](/key-concepts/communities/overview) | [Communities Deep Dive](/core-concepts/communities) | [API Reference](/api-reference/endpoint/create-community)
</Accordion></div>
  <div data-glossary-item data-term="Connection Type"><Accordion title="Connection Type">
The transport channel used to connect an agent to Bluejay for simulations. Supported types include telephony, SIP, LiveKit, WebSocket, and HTTP.

See more: [Agents Overview](/key-concepts/agents/overview) | [Simulation Types](/test/simulations/types)
</Accordion></div>
  <div data-glossary-item data-term="Custom Metric"><Accordion title="Custom Metric">
A user-defined evaluation signal used to measure what matters for a specific use case. Custom metrics can be LLM-judged (scored via a natural-language prompt) or formula-based (computed from other metrics), and they work identically across simulations and production observability.

See more: [Custom Metrics Overview](/key-concepts/custom-metrics/overview) | [API Reference](/api-reference/endpoint/create-custom-metric)
</Accordion></div>
  <div data-glossary-item data-term="Customer Persona"><Accordion title="Customer Persona">
A reusable profile that combines customer traits into a template for Digital Human generation. Personas can be created via the API and attached to agents.

See more: [Customer Traits](/key-concepts/customer-traits/overview) | [API Reference](/api-reference/endpoint/create-customer-persona)
</Accordion></div>
  <div data-glossary-item data-term="Customer Traits"><Accordion title="Customer Traits">
Structured behavioral and contextual attributes — such as tone, language, urgency, and simulated PII — that shape how a Digital Human behaves during a simulation.

See more: [Customer Traits Overview](/key-concepts/customer-traits/overview) | [Digital Humans](/key-concepts/digital-humans/overview)
</Accordion></div>
</section>

</Accordion></div>
  <div data-glossary-item data-term="Digital Human"><Accordion title="Digital Human">
A synthetic customer used to simulate realistic conversations during evaluation. Each Digital Human has a persona, scenario script, customer traits, and success criteria. They can be created manually or auto-generated.

A tag (e.g. &quot;production&quot;, &quot;staging&quot;) applied to prompt or knowledge base versions for environment management and version pinning across simulation runs.

</Accordion></div>
  <div data-glossary-item data-term="Latency"><Accordion title="Latency">
The time elapsed between the end of a user's utterance and the start of the agent's spoken response. Latency is a key voice AI quality signal — high latency degrades perceived naturalness. Bluejay captures and surfaces latency metrics on call logs and dashboards.
</Accordion></div>
  <div data-glossary-item data-term="LiveKit"><Accordion title="LiveKit">
A WebRTC-based integration for room-based voice simulations. Bluejay creates a LiveKit room and token, then a Digital Human joins the room alongside the agent for real-time voice testing.

See more: [LiveKit Integration](/simulation-integrations/livekit) | [Simulation Types](/test/simulations/types)
</Accordion></div>
  <div data-glossary-item data-term="LLM-as-a-Judge"><Accordion title="LLM-as-a-Judge">
A custom metric mode where a natural-language prompt is used to score conversations via an LLM. The prompt describes what to evaluate and how to score, and the LLM returns a structured result.

See more: [Custom Metrics](/key-concepts/custom-metrics/overview)
</Accordion></div>
</section>

  <div data-glossary-item data-term="Regression Testing"><Accordion title="Regression Testing">
Re-running simulations after agent changes (prompt updates, model swaps, configuration tweaks) to detect quality degradation before shipping to production.

</Accordion></div>
  <div data-glossary-item data-term="Simulation"><Accordion title="Simulation">
A controlled pre-production evaluation container that groups Digital Humans (and optionally Communities) to test agent behavior against realistic scenarios. Simulations are re-runnable and scored with custom metrics.

  <div data-glossary-item data-term="Simulation Run"><Accordion title="Simulation Run">
