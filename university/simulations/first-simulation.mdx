## Objectives

- Create a new Simulation attached to an Agent
- Add a Digital Human to the Simulation
- Run the Simulation and review results

## Walkthrough

<Steps>
  <Step title="Navigate to Simulations">
    Open the **Simulations** page from the sidebar and click **New Simulation**. Select the Agent you created in Course 1.
  </Step>
  <Step title="Configure the scenario">
    Give your Simulation a descriptive name (e.g., "Happy-path order placement"). Set the connection type that matches your agent — telephony, SIP, LiveKit, websocket, or HTTP.

    For a full breakdown of connection types, see [Simulation Integrations](/simulation-integrations/livekit).
  </Step>
  <Step title="Add a Digital Human">
    Click **Add Digital Human** and fill in the persona details: name, personality traits, and the scenario script that describes what the caller wants to accomplish. The scenario script is the most important field — it tells the Digital Human *what* to do during the call.
  </Step>
  <Step title="Attach Custom Metrics">
    Select or create Custom Metrics that define success for this conversation. For example: "Did the agent confirm the order?" or "Was the caller greeted within 5 seconds?"
  </Step>
  <Step title="Run the Simulation">
    Click **Run** and watch the conversation unfold in real time. When it finishes, review the transcript and metric scores on the results page.
  </Step>
</Steps>

## Knowledge check

<AccordionGroup>
  <Accordion title="What is a scenario script?">
    A scenario script is a natural-language description of the Digital Human's goal, constraints, and behavior during a simulated conversation. It drives what the synthetic caller says and does.
  </Accordion>
  <Accordion title="Can you run a Simulation more than once?">
    Yes. Simulations are designed to be repeatable. Each execution creates a new **Simulation Run** with its own transcript and scores, so you can track changes over time.
  </Accordion>
</AccordionGroup>
