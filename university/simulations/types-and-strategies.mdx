## Objectives

- Distinguish between the available simulation types
- Choose the right type for your testing scenario
- Design a multi-simulation test plan

## Simulation types at a glance

Bluejay supports several simulation types, each suited to different testing needs. For the full reference, see [Simulation Types](/test/simulations/types).

| Type | Best for |
|---|---|
| **Voice (Telephony / SIP)** | End-to-end testing of phone-based agents |
| **LiveKit** | Real-time WebRTC voice agents |
| **Websocket** | Streaming text or audio agents |
| **HTTP Webhook** | Request/response chat agents |

## Building a test plan

A strong test plan combines multiple simulations to cover:

1. **Happy paths** — the most common, expected interactions
2. **Edge cases** — ambiguous inputs, interruptions, silence, long pauses
3. **Failure modes** — what happens when the agent can't fulfill a request
4. **Regression checks** — re-running known scenarios after prompt or model changes

<Tip>
Use **Communities** to maintain a stable pool of Digital Humans across simulations so results are comparable over time.
</Tip>

## Knowledge check

<AccordionGroup>
  <Accordion title="When should you use an HTTP Webhook simulation?">
    When your agent communicates via synchronous HTTP request/response rather than a persistent voice or streaming connection.
  </Accordion>
  <Accordion title="Why run the same simulation multiple times?">
    LLM-based agents are non-deterministic. Running the same scenario multiple times helps you measure consistency and catch intermittent failures.
  </Accordion>
</AccordionGroup>
