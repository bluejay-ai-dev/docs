## Objectives

- Map out the relationship between Agents, Simulations, Digital Humans, and Communities
- Understand what Custom Metrics are and why they matter
- Know where each entity lives in the dashboard

## Key entities

| Entity | What it represents | Dashboard location |
|---|---|---|
| **Agent** | A mirror of your production conversational AI | Agents page |
| **Simulation** | A test scenario that runs against an Agent | Simulations page |
| **Digital Human** | A synthetic persona that plays the caller/customer role | Within a Simulation |
| **Community** | A reusable group of Digital Humans | Communities page |
| **Custom Metric** | An evaluation criterion scored after each conversation | Custom Metrics page |

## How they connect

```
Agent
 ├── Simulation
 │    ├── Digital Human (inline)
 │    └── Community (shared pool)
 └── Custom Metrics (attached for evaluation)
```

An **Agent** can have many **Simulations**. Each Simulation defines the scenario and attaches one or more **Digital Humans** — either created inline or pulled from a **Community**. After a conversation completes, **Custom Metrics** score the result.

For full reference material on each entity, explore the [Key Concepts](/key-concepts/agents/overview) section.

## Knowledge check

<AccordionGroup>
  <Accordion title="Can a Digital Human belong to multiple Communities?">
    Yes. Digital Humans are standalone entities that can be added to any number of Communities, making it easy to reuse personas across different Simulations.
  </Accordion>
  <Accordion title="When are Custom Metrics evaluated?">
    Custom Metrics are evaluated after a conversation ends — whether that conversation was a simulation run or a production call ingested via observability.
  </Accordion>
</AccordionGroup>
