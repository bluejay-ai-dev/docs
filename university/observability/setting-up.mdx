## Objectives

- Understand the observability data flow
- Send your first production call to Bluejay via the API
- Verify that evaluation results appear in the dashboard

## How observability works

```
Your Agent (production)
  │
  ├── call completes → transcript + metadata
  │
  └── POST /evaluate → Bluejay
       │
       ├── runs Custom Metrics
       └── stores results → Dashboard
```

Bluejay's observability pipeline ingests completed conversations, runs your Custom Metrics against them, and surfaces the results on dashboards and in alerts.

## Walkthrough

<Steps>
  <Step title="Choose your integration method">
    Bluejay supports multiple ways to ingest production data:
    - **API integration** — send transcripts directly via the `/evaluate` endpoint
    - **Provider integration** — connect Retell, Vapi, Bland, ElevenLabs, or other supported providers

    See the [Observability API Integration Tutorial](/monitor/observability/api-integration-tutorial) for step-by-step API instructions, or browse [Integrations](/simulation-integrations/livekit) for provider-specific guides.
  </Step>
  <Step title="Send a test call">
    Using the API or your chosen provider integration, send a single completed conversation to Bluejay. Confirm it appears on the **Observability** page.
  </Step>
  <Step title="Review evaluation results">
    Click into the call log entry to see the transcript, metadata, and Custom Metric scores. Verify the metrics match your expectations.
  </Step>
</Steps>

## Knowledge check

<AccordionGroup>
  <Accordion title="What data does Bluejay need for observability?">
    At minimum, a transcript and the agent identifier. Metadata like call duration, caller info, and timestamps enriches the evaluation but is optional.
  </Accordion>
  <Accordion title="Can you re-evaluate a call with different metrics?">
    Yes. Use the **Re-evaluate** action on any call log to re-run it against updated or additional Custom Metrics.
  </Accordion>
</AccordionGroup>
