Get from account creation to running simulations, evaluating production calls, and optimizing your agent quality in under 15 minutes.

## Prerequisites

- A Bluejay account ([book a demo](https://cal.com/rohanv/15min) if you don't have one)
- For API steps: an API key from the [API Keys page](https://app.getbluejay.ai/api-keys)

  <Tab title="UI">

    ## Basic

    Everything you need to run your first simulation and connect production monitoring.

    <Steps>
      <Step title="Add an Agent">
        An agent represents the conversational AI system you want to test and monitor. Navigate to the **Agents** page in the Bluejay dashboard and click **Add Agent**. Fill in the agent name, system prompt, knowledge base, and goals.

        {/* screenshot: add-agent-ui */}
      </Step>

      <Step title="Create a Simulation">
        A simulation groups digital humans together so you can run them in parallel against your agent â€” think of it as a test suite. Open your agent and click **Create Simulation**. Give it a name and optionally configure settings like max call duration or sequential calling.

        {/* screenshot: create-simulation-ui */}
      </Step>

      <Step title="Create or Generate Digital Humans">
        Digital humans are the synthetic callers that interact with your agent during simulations. Inside your simulation, click **Generate Digital Humans**. Choose from goal adherence, red teaming, load testing, or customer persona modes. You can also manually create individual digital humans for specific scenarios.

        {/* screenshot: generate-digital-humans-ui */}
      </Step>

      <Step title="Run the Simulation">
        Click the **Run** button on your simulation. Bluejay's digital humans will call your agent, and results are evaluated automatically. You can watch conversations happen in real time and review results as they complete.

        {/* screenshot: run-simulation-ui */}
      </Step>

      <Step title="Add Custom Metrics">
        Custom Metrics define the exact quality signals you care about â€” compliance checks, empathy scoring, task completion, or anything specific to your domain. Navigate to the **Custom Metrics** section of your agent, click **Create Metric**, and define the name, description, response type, and scoring guidance.

        {/* screenshot: create-custom-metric-ui */}
      </Step>

      <Step title="Hook Up Observability">
        Observability evaluates your production conversations against the same Custom Metrics you use in simulations. Go to the **Observability** tab for your agent. You can connect a native integration (Retell, Vapi, Bland, ElevenLabs) or configure webhook-based ingestion directly from the dashboard.

        {/* screenshot: observability-setup-ui */}
      </Step>
    </Steps>

    ## Advanced

    Level up with alerts, dashboards, optimization, automated testing, and prompt versioning.

    <Steps>
      <Step title="Create Alerts">
        Alerts notify your team when a metric crosses a threshold so issues get caught before they compound. Navigate to the **Alerts** configuration for your agent. Set the metric, threshold, and delivery channel (Slack, email).

        {/* screenshot: create-alert-ui */}

        Learn more in the [Alerts overview](/key-concepts/alerts/overview).
      </Step>

      <Step title="Create Custom Dashboards">
        Dashboards aggregate simulation and production data into a single view. Open the **Dashboards** section to see at-a-glance health scores, trend sparklines, and alert badges across all your agents. Customize views to focus on the metrics that matter most.

        {/* screenshot: custom-dashboard-ui */}

        Learn more in the [Dashboards overview](/key-concepts/dashboards/overview).
      </Step>

      <Step title="Optimize Metrics via Metrics Lab">
        Metrics Lab lets you prototype, test, and refine Custom Metrics before deploying them. Open **Metrics Lab** from the sidebar. Import or draft Custom Metrics, annotate sample conversations, and run side-by-side comparisons to find the most reliable scoring approach. Promote validated metrics to production with one click.

        {/* screenshot: metrics-lab-ui */}

        Learn more in the [Metrics Lab overview](/key-concepts/metrics-lab/overview).
      </Step>

      <Step title="Test Agents via Workflows">
        Workflows let you chain simulation runs, evaluations, and notifications into automated pipelines. Navigate to **Workflows** and create a new workflow. Use the visual builder to add steps â€” simulation runs, evaluations, and notification nodes â€” and connect them into a pipeline. Set a schedule to run automatically.

        {/* screenshot: create-workflow-ui */}

        Learn more in the [Workflows overview](/key-concepts/workflows/overview).
      </Step>

      <Step title="Version Prompts via Prompt Versioning">
        Prompt Versioning tracks changes to your agent's system prompt over time. Open the **Prompts** section of your agent. Create a new version with an updated prompt, add a commit message, and optionally tag it with labels like `production` or `staging`. Compare performance across simulation runs and roll back when needed.

        {/* screenshot: prompt-versioning-ui */}
      </Step>
    </Steps>

    ## Basic

    Everything you need to run your first simulation and connect production monitoring.

    ```python
    import requests

    API_KEY = "your-api-key"
    BASE_URL = "https://api.getbluejay.ai/v1"
    ```

    <Steps>
      <Step title="Add an Agent">
        An agent represents the conversational AI system you want to test and monitor. You'll attach simulations, metrics, and observability pipelines to it.

        ```python
        agent = requests.post(
            f"{BASE_URL}/add-agent",
            headers={"X-API-Key": API_KEY},
            json={
                "name": "Support Agent",
                "system_prompt": "You are a helpful customer support agent for Acme Corp.",
                "knowledge_base": "Acme Corp FAQ and product documentation.",
                "goals": [
                    "Resolve customer issues accurately",
                    "Maintain a professional and empathetic tone",
                    "Escalate when unable to resolve"
                ],
                "type": "INBOUND",
                "mode": "VOICE"
            }
        ).json()

        agent_id = agent["agent_id"]
        print(f"Agent created: {agent_id}")
        ```

        <Note>
          Save the `agent_id` from the response â€” you'll use it in every subsequent step.
        </Note>

        See the full [Add Agent](/api-reference/endpoint/add-agent) reference for all available fields like `connection_type`, `phone_number`, and `external_agent_id`.
      </Step>

      <Step title="Create a Simulation">
        A simulation groups digital humans together so you can run them in parallel against your agent â€” think of it as a test suite.

        ```python
        simulation = requests.post(
            f"{BASE_URL}/create-simulation",
            headers={"X-API-Key": API_KEY},
            json={
                "agent_id": str(agent_id),
                "name": "Support Agent â€” Regression Suite",
                "max_call_duration": 5,
                "max_call_duration_units": "minutes"
            }
        ).json()

        simulation_id = simulation["simulation_id"]
        print(f"Simulation created: {simulation_id}")
        ```

        See the full [Create Simulation](/api-reference/endpoint/create-simulation) reference for options like `sequential_calling`, `max_concurrent`, and `selected_custom_metrics`.
      </Step>

      <Step title="Create or Generate Digital Humans">
        Digital humans are the synthetic callers that interact with your agent during simulations. You can generate them automatically from your agent's goals or create them manually for specific scenarios.

        **Generate automatically from goals:**

        ```python
        digital_humans = requests.post(
            f"{BASE_URL}/generate-digital-humans",
            headers={"X-API-Key": API_KEY},
            json={
                "agent_id": str(agent_id),
                "simulation_id": str(simulation_id),
                "goal_adherence": [
                    {"goal": "Resolve billing dispute", "num_calls": 3},
                    {"goal": "Process a return request", "num_calls": 2}
                ],
                "red_teaming": 2
            }
        ).json()

        print(f"Generated {len(digital_humans['digital_humans'])} digital humans")
        ```

        **Or create one manually:**

        ```python
        dh = requests.post(
            f"{BASE_URL}/create-digital-human",
            headers={"X-API-Key": API_KEY},
            json={
                "simulation_id": str(simulation_id),
                "name": "Frustrated Customer",
                "scenario": "Customer received the wrong item and wants a full refund.",
                "success_criteria": "Agent processes the return and confirms refund timeline."
            }
        ).json()
        ```

        See [Generate Digital Humans](/api-reference/endpoint/generate-digital-humans) and [Create Digital Human](/api-reference/endpoint/create-digital-human) for all options.
      </Step>

      <Step title="Run the Simulation">
        Queue a simulation run to start the conversations. Bluejay's digital humans will call your agent, and results are evaluated automatically.

        ```python
        run = requests.post(
            f"{BASE_URL}/queue-simulation-run",
            headers={"X-API-Key": API_KEY},
            json={
                "simulation_id": str(simulation_id)
            }
        ).json()

        print(f"Simulation run queued: {run['simulation_run_id']}")
        print(f"Result IDs: {run['simulation_result_ids']}")
        ```

        <Tip>
          Use the [Get Simulation Runs](/api-reference/endpoint/get-simulation-runs) endpoint to poll for completion, or set up a [webhook](/api-reference/webhook/events-webhook) to get notified when runs finish.
        </Tip>

        See the full [Queue Simulation Run](/api-reference/endpoint/queue-simulation-run) reference for options like `phone_number`, `digital_human_ids`, and `prompt_id`.
      </Step>

      <Step title="Add Custom Metrics">
        Custom Metrics define the exact quality signals you care about â€” compliance checks, empathy scoring, task completion, or anything specific to your domain.

        ```python
        metric = requests.post(
            f"{BASE_URL}/create-custom-metric",
            headers={"X-API-Key": API_KEY},
            json={
                "agent_ids": [agent_id],
                "name": "Identity Verification",
                "description": "Did the agent verify the customer's identity before sharing account details?",
                "response_type": "pass_fail",
                "scoring_guidance": "Pass if the agent asks for at least two identifying pieces of information before disclosing any account-specific data. Fail otherwise."
            }
        ).json()

        print(f"Metric created: {metric['id']}")
        ```

        See [Create Custom Metric](/api-reference/endpoint/create-custom-metric) for all response types (`pass_fail`, `yes_no`, `qualitative`, `quantitative`, `json`, `enum`).
      </Step>

      <Step title="Hook Up Observability">
        Observability evaluates your production conversations against the same Custom Metrics you use in simulations. Send calls to Bluejay via the API, webhooks, or native integrations.

        ```python
        evaluation = requests.post(
            f"{BASE_URL}/evaluate",
            headers={"X-API-Key": API_KEY},
            json={
                "agent_id": str(agent_id),
                "transcript": [
                    {"speaker": "CUSTOMER", "utterance": "Hi, I need help with my recent order."},
                    {"speaker": "AGENT", "utterance": "Of course! Could you provide your order number?"},
                    {"speaker": "CUSTOMER", "utterance": "It's 12345."},
                    {"speaker": "AGENT", "utterance": "I see order 12345. It looks like it shipped yesterday. You should receive it by Friday."}
                ],
                "participants": [
                    {"role": "AGENT", "spoke_first": False, "name": "Support Agent"},
                    {"role": "CUSTOMER", "spoke_first": True, "name": "Customer"}
                ],
                "call_direction": "INBOUND"
            }
        ).json()

        print(f"Evaluation queued: {evaluation['call_id']}")
        ```

        You can also send audio recordings via `recording_url` or `recording_base64` instead of transcripts. See the full [Evaluate](/api-reference/endpoint/evaluate) reference.

        | Ingestion Method | Best For |
        |---|---|
        | [Evaluate API](/api-reference/endpoint/evaluate) | Direct backend integration |
        | [Webhook ingestion](/core-concepts/webhook) | Platforms with outbound webhooks |
        | [Retell](/integrations/retell) | Retell-powered agents |
        | [Vapi](/integrations/vapi) | Vapi-powered agents |
        | [Bland](/integrations/bland) | Bland-powered agents |
        | [ElevenLabs](/integrations/eleven-labs) | ElevenLabs Conversational AI |
      </Step>
    </Steps>

    ## Advanced

    Level up with alerts, dashboards, optimization, automated testing, and prompt versioning.

    <Steps>
      <Step title="Create Alerts">
        Alerts notify your team when a metric crosses a threshold so issues get caught before they compound. Alerts are configured through the Bluejay dashboard â€” set threshold-based triggers on any Custom Metric and route notifications to Slack channels or email.

        <Note>
          Connect Bluejay to Slack via the [Slack integration](/integrations/slack) to receive alert notifications in your team channels.
        </Note>

        Learn more in the [Alerts overview](/key-concepts/alerts/overview).
      </Step>

      <Step title="Create Custom Dashboards">
        Dashboards aggregate simulation and production data into a single view â€” health scores, trend sparklines, and alert badges for every agent. Dashboards are configured through the Bluejay UI and pull from both simulation results and observability evaluations, so once you've completed the basic steps, your dashboards will populate automatically.

        Learn more in the [Dashboards overview](/key-concepts/dashboards/overview).
      </Step>

      <Step title="Optimize Metrics via Metrics Lab">
        Metrics Lab lets you prototype, test, and refine Custom Metrics before deploying them. It's an interactive UI experience â€” import or draft metrics, annotate sample conversations, run side-by-side comparisons, and promote validated metrics to production. Once refined, you can manage metrics programmatically via the [Custom Metrics API](/api-reference/endpoint/create-custom-metric).

        Learn more in the [Metrics Lab overview](/key-concepts/metrics-lab/overview).
      </Step>

      <Step title="Test Agents via Workflows">
        Workflows let you chain simulation runs, evaluations, and notifications into automated pipelines. Schedule them on a cron cadence to run nightly regression suites or weekly quality checks.

        ```python
        workflow = requests.post(
            f"{BASE_URL}/workflows",
            headers={"X-API-Key": API_KEY},
            json={
                "name": "Nightly Regression Suite",
                "agent_ids": [agent_id],
                "workflow": {
                    "nodes": [
                        {
                            "id": "start",
                            "type": "start",
                            "data": {"label": "Start", "description": "Begin nightly regression"}
                        },
                        {
                            "id": "run_sim",
                            "type": "simulation",
                            "data": {"label": "Run Simulation", "description": "Execute regression suite"}
                        },
                        {
                            "id": "evaluate",
                            "type": "evaluate",
                            "data": {"label": "Evaluate", "description": "Score results"}
                        }
                    ],
                    "edges": [
                        {"id": "e1", "source": "start", "target": "run_sim"},
                        {"id": "e2", "source": "run_sim", "target": "evaluate"}
                    ]
                },
                "enabled_for_evaluation": True
            }
        ).json()

        print(f"Workflow created: {workflow['id']}")
        ```

        You can also attach a cron schedule via the [Create Schedule](/api-reference/endpoint/create-schedule) endpoint. See [Create Workflow](/api-reference/endpoint/create-workflow) for the full reference.
      </Step>

      <Step title="Version Prompts via Prompt Versioning">
        Prompt Versioning tracks changes to your agent's system prompt over time. Create labeled versions, compare performance across simulation runs, and roll back when needed.

        ```python
        prompt_version = requests.post(
            f"{BASE_URL}/agents/{agent_id}/prompts/versions",
            headers={"X-API-Key": API_KEY},
            json={
                "prompt_text": "You are a helpful customer support agent for Acme Corp. Always verify the customer's identity before sharing account details.",
                "commit_message": "Added identity verification requirement",
                "labels": ["staging"]
            }
        ).json()

        print(f"Prompt version {prompt_version['version']} created (id: {prompt_version['id']})")
        ```

        <Tip>
          Pass a `prompt_id` when [queuing a simulation run](/api-reference/endpoint/queue-simulation-run) to test a specific prompt version against your existing digital humans.
        </Tip>

        See [Create Prompt Version](/api-reference/endpoint/create-prompt-version) for the full reference.
      </Step>
    </Steps>
