When an ElevenLabs agent is synced into Bluejay, its multi-agent workflow renders as an editable graph in the **Workflows** tab of Agent Settings. You can edit any node from the canvas, push your edits back to ElevenLabs, and generate digital humans that exercise every distinct path through the flow.

The walkthrough below covers the entire simulation flow end to end, from opening the Workflows tab through running the call and reviewing the result.

<Note>
  For the production-call observability integration (webhook setup for real calls), see [ElevenLabs Observability](/integrations/eleven-labs).
</Note>

### Prerequisites

- An ElevenLabs agent that uses a multi-agent workflow
- That agent synced into Bluejay (see [ElevenLabs Observability](/integrations/eleven-labs) for the connection steps)

### 1. The Workflows Tab

Open Agent Settings, then click the **Workflows** tab. Your synced ElevenLabs workflow renders as a graph on the canvas, using the same controls as Bluejay's Scenario Builder.

Three actions live in the top-right of the canvas:

- **Sync** pulls the latest workflow from ElevenLabs and replaces your local copy.
- **Push** sends your edits back to ElevenLabs.
- **Branches** lets you switch between workflow branches and merge back to main.

![The Workflows tab in Bluejay](/images/elevenlabs-sim-workflows-tab.gif)

#### Editing nodes inline

Click any node to open the side panel and edit it without leaving Bluejay:

- **General** controls voice, model, eagerness, spelling patience, speculative turn, and auto-advance.
- **Tools** toggles the built-in tools and lets you add or remove custom tools.
- **Edges** sets the condition that takes the agent from this step to the next.

With a node selected, the inline **Duplicate** and **Delete** actions appear directly on the node. Press **Cmd+Z** to undo a canvas edit and **Cmd+Shift+Z** to redo it.

#### Drawing an edge between two nodes

1. An empty **Add condition** pill appears when you drag a new edge between two nodes.
2. Click the **Add condition** pill and type the trigger that takes the agent from the first node to the next.

### 2. Generate From Workflow

Once your workflow is synced, you can generate digital humans that cover every distinct path through it.

1. On the agent, click **Create Simulation** to open the **Create Simulation** dialog.
2. Choose **Generate From Workflow** as the simulation type.
3. Bluejay enumerates every distinct path through the synced workflow graph and lists the paths as numbered step pills.
4. Choose how many digital humans to generate per path. For example, three paths times four digital humans equals twelve scenarios.
5. Click **Generate** to create the digital humans. Each one receives an intent that drives the agent down its assigned path.
6. Open the **Persona** tab and confirm the path label stays attached to each digital human, directly under the success criteria, so you can verify the route the digital human is covering.

![Generate digital humans from a synced workflow](/images/elevenlabs-sim-generate.gif)

### 3. Run your Simulation

1. In the persona list, select all the digital humans that were just generated.
2. Click **Run** to dispatch the calls. Bluejay runs one simulation per selected digital human.
3. When the runs finish, the simulation result opens with the transcript, tool calls, and metric scores for each call. Review the results the same way you would for any other simulation in Bluejay.

![Run a simulation against the generated digital humans](/images/elevenlabs-sim-run.gif)
