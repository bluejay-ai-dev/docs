Bluejay has first-class support for ElevenLabs agent workflows. When you add an ElevenLabs agent, Bluejay pulls in its multi-agent workflow and shows it as an editable graph in the **Workflows** tab of Agent Settings. Edit nodes inline, push your changes back to ElevenLabs, and generate digital humans that test every path through the flow.

<Note>
  For the Observability integration (webhooks for production calls), see [ElevenLabs Observability](/integrations/eleven-labs).
</Note>

## The Workflows Tab

Open **Agent Settings → Workflows** to see your synced workflow as a graph. The canvas matches the Scenario Builder look-and-feel so it reads the same regardless of which feature you're in.

Three actions live in the top-right of the canvas:

- **Sync** to pull the latest workflow from ElevenLabs and replace your local copy.
- **Push** to send your edits back to ElevenLabs.
- **Branches** to switch between your workflow's branches and merge back to main.

### Editing Nodes Inline

Click any node to open the side panel and edit it without leaving Bluejay:

- **General**: voice, model, eagerness, spelling patience, speculative turn, auto-advance.
- **Tools**: toggle the built-in tools and add or remove custom tools.
- **Edges**: set the condition that takes the agent from this step to the next.

Drawing a new edge between two nodes drops an empty **Add condition** pill on the connection. Click it to type the trigger. Duplicate and delete actions appear inline on the selected node. Cmd+Z / Cmd+Shift+Z undo and redo your canvas edits.

## Generate From Workflow

Once your workflow is synced, the **Generate From Workflow** simulation type becomes available in the Create Simulation dialog for that agent.

Bluejay finds every distinct path through the graph and shows them as numbered step pills. You choose how many digital humans to generate per path. Three paths × four digital humans = twelve scenarios.

Each generated digital human gets an intent that naturally drives the agent down its assigned path, plus the path it's testing right under the success criteria on the Persona tab. The strand sticks around so you can always see which route a given digital human is covering.
