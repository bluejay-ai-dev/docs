## Update workflow (v2) — PUT /v1/workflows-v2/{workflow_id}

> **What this endpoint does:** Partial update of name, description, `agent_ids`, and/or `definition`.

**Endpoint:** PUT `https://api.getbluejay.ai/v1/workflows-v2/{workflow_id}`
**Auth:** `Authorization: Bearer` (JWT) or `X-API-Key` header
**Content-Type:** application/json

### Important semantics
- Omitted fields are left unchanged.
- If `agent_ids` appears in the body (including `[]`), all agent junction links are **replaced**; `[]` clears links.
- If `definition` appears, it is **fully replaced** and validated like create.

When you send `definition`, it must use the same discriminated types as **POST** `/v1/workflows-v2`: `WorkflowV2Node` (by node `type`), `WorkflowV2SingleNodeData` / `WorkflowV2OptionsNodeData`, and `WorkflowV2BranchOption` for options branches. See [Create workflow (v2)](/api-reference/endpoint/create-workflow-v2) for validation rules and examples.

### Example
```python
import requests

def update_workflow_v2(workflow_id: str, payload: dict, api_key: str) -> dict:
    url = f"https://api.getbluejay.ai/v1/workflows-v2/{workflow_id}"
    headers = {"X-API-Key": api_key}
    response = requests.put(url, headers=headers, json=payload)
    response.raise_for_status()
    return response.json()
```
