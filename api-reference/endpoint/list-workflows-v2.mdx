## List workflows (v2) — GET /v1/workflows-v2

> **What this endpoint does:** List workflows v2 for the authenticated user's organization, optionally filtered by `agent_id`.

**Endpoint:** GET `https://api.getbluejay.ai/v1/workflows-v2`
**Auth:** `X-API-Key` header

Each workflow in `workflows` includes `definition` with the same OpenAPI typing as create: `WorkflowV2Node` (by graph `type`), `WorkflowV2SingleNodeData` (by `data.type`), `WorkflowV2OptionsNodeData`, and `WorkflowV2BranchOption`. See also [Create workflow (v2)](/api-reference/endpoint/create-workflow-v2).

### Query parameters
| Name | Type | Description |
|------|------|-------------|
| agent_id | integer (optional) | When set, only workflows linked to this agent |

### Example
```python
import requests

def list_workflows_v2(api_key: str, agent_id: int | None = None) -> dict:
    url = "https://api.getbluejay.ai/v1/workflows-v2"
    headers = {"X-API-Key": api_key}
    params = {}
    if agent_id is not None:
        params["agent_id"] = agent_id
    response = requests.get(url, headers=headers, params=params or None)
    response.raise_for_status()
    return response.json()
```
