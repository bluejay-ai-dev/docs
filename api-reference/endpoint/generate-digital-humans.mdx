## Generate Digital Humans — POST /v1/generate-digital-humans

> **What this endpoint does:** Given an agent this endpoint generates digital humans to test that agent based on varying simulation types. **Per scenario:** Non-empty `enriched_playback` forces `description` and `success_criteria` to empty strings. If scenario tags match workflow playback (tag lowercased contains `workflow`), the same clearing applies and `role_description` is forced to null (even without enriched_playback). **Batch:** Objectives / intent-summary generation runs only for created rows with non-empty description — workflow-style rows with empty description are skipped. **Graph workflows (`workflow_adherence_v2`):** Each value in the dictionary is the number of paths to generate for that workflow. Counts are silently capped at **200 paths per workflow** — values above 200 are treated as 200, not rejected. `auto_gen_workflow_v2_adherence_scenarios` feeds this endpoint when `workflow_adherence_v2` is used (empty description/success_criteria, filled enriched_playback).

**Endpoint:** POST `https://api.getbluejay.ai/v1/generate-digital-humans`
**Auth:** `X-API-Key` header
**Content-Type:** application/json
