# Integrate Create Custom Metric — POST /v1/create-custom-metric

**You are a senior backend engineer. Your task: integrate the endpoint below into the existing codebase with minimal, surgical changes. Do not refactor, rename, or restructure anything beyond what is required for this integration.**

## Endpoint Details
- **Method:** POST
- **URL:** `https://api.getbluejay.ai/v1/create-custom-metric`
- **Auth:** API Key via `X-API-Key` header
- **Content-Type:** application/json

## Parameters
| Name | Location | Type | Required | Description |
|------|----------|------|----------|-------------|
| X-API-Key | header | string | yes | API key required to authenticate requests. |
| agent_id | body | integer (nullable) | no | ID of the agent this metric belongs to (deprecated, use agent_ids instead) |
| agent_ids | body | array[integer] (nullable) | no | List of agent IDs to associate this metric with |
| name | body | string | yes | Name of the custom metric |
| description | body | string | yes | Description of what this metric measures |
| response_type | body | string (enum: pass_fail, yes_no, qualitative, quantitative, json, enum) | yes | Enum representing the possible response types for custom metrics. |
| min_value | body | number (nullable) | no | Minimum value for quantitative metrics |
| max_value | body | number (nullable) | no | Maximum value for quantitative metrics |
| category | body | string (nullable) | no | Category for organizing metrics |
| tags | body | array[string] (nullable) | no | Tags for categorizing the metric |
| scoring_guidance | body | string (nullable) | no | Guidance on how to score this metric |
| eval_route | body | string (enum: AUDIO, TEXT, AUTO) (nullable) | no | Route for evaluating the metric (default: AUTO) |
| model | body | string (nullable) | no | Model to use for evaluating the metric |
| temperature | body | number (nullable) | no | Temperature for evaluating the metric |
| enum_options | body | array[string] (nullable) | no | Options for enum metrics |
| allow_not_applicable | body | boolean | no | Whether this metric allows 'not applicable' responses. (default: False) |
