Many teams build voice agents as **multi-agent workflows**: a root agent that hands off to specialised sub-agents as the conversation progresses (e.g. *Greeting* → *Gather Details* → *Resolve*). Bluejay can pull that workflow off your provider, store it on the agent record, render it as an editable graph, and use it to generate digital humans that test every path through the flow.

## What You'll Learn

- What an agent workflow is and how Bluejay stores it
- What the Workflow tab in Agent Settings lets you do
- How workflows power the **Generate From Workflow** simulation type

## How It Works

Each Bluejay agent has two related fields that drive the Workflow feature:

- **`provider`** — the source platform the agent's workflow comes from (e.g. `ELEVENLABS`). Tells Bluejay how to parse the workflow JSON and which integration to use for sync / push.
- **`workflow`** — the provider's raw graph stored as JSON. Bluejay treats this as the source of truth for the agent's flow.

The **Workflow** tab in Agent Settings only appears for agents whose `provider` supports workflow integration. For unsupported providers (including the default `CUSTOM` single-prompt agent) the tab is hidden.

## What The Workflow Tab Gives You

- **Visualise** — render the synced workflow as a node-and-edge graph in the same look-and-feel as the Scenario Builder canvas.
- **Edit** — open any node to change its name, system prompt, voice, LLM, eagerness, spelling patience, speculative-turn, auto-advance, and tool toggles. Click an edge to set its transition condition inline.
- **Sync from provider** — pull the latest workflow off the provider, overwriting the local copy.
- **Push to provider** — send local edits back through the provider's update endpoint so the canonical version stays in lockstep.
- **Branches** — if your provider supports versioned branches of a workflow, switch between them and merge edits up to main.

Sync replaces the local workflow blob wholesale. Push replaces the provider copy wholesale. There is no field-level merge — pick one source of truth at a time.

## Driving Simulations

The **Generate From Workflow** simulation type enumerates every distinct path through the synced workflow (Start → step → step → …) and generates digital humans whose intents naturally drive the agent down each path. Each path becomes its own digital human card with the path strand visible underneath the success criteria in the Persona tab.

You pick how many digital humans to generate per path. Three discovered paths × four digital humans = twelve scenarios, each tailored to drive its assigned path — generated in parallel, one LLM call per path.
