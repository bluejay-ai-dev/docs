Many teams build voice agents as **multi-agent workflows**: a root agent that hands off to specialised sub-agents as the conversation progresses (e.g. *Greeting* → *Gather Details* → *Resolve*). Bluejay pulls that workflow in from your provider, renders it as an editable graph, and uses it to generate digital humans that test every path through the flow.

## What You'll Learn

- What an agent workflow is
- What the Workflows tab in Agent Settings lets you do
- How workflows power the **Generate From Workflow** simulation type

## Actions

The **Workflows** tab in Agent Settings appears for any agent whose provider supports workflows. From there you can:

- **Visualise** the workflow as a node-and-edge graph.
- **Edit** any node (name, system prompt, voice, model, tools) or click an edge to set its transition condition.
- **Sync** to pull the latest workflow from your provider.
- **Push** to send your edits back to the provider.
- **Branches** to switch between branches and merge back to main, if your provider supports them.

Sync and Push each replace the destination wholesale, so pick one source of truth at a time.

## Driving Simulations

The **Generate From Workflow** simulation type enumerates every distinct path through the synced workflow (Start → step → step → …) and generates digital humans whose intents naturally drive the agent down each path. Each path becomes its own digital human card with the path strand visible underneath the success criteria in the Persona tab.

You pick how many digital humans to generate per path. Three discovered paths × four digital humans = twelve scenarios, each tailored to drive its assigned path.
